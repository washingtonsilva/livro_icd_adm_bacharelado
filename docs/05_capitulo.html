6  Capítulo 5 — RStudio e fluxo de trabalho

Este capítulo dá o primeiro passo para um trabalho organizado e reproduzível com as ferramentas que utilizaremos: entender o RStudio, configurar o ambiente e adotar boas práticas desde o início.

NotaObjetivos do capítulo
  • Reconhecer o papel do RStudio como IDE para o R.
  • Navegar pela interface e usar os painéis principais.
  • Instalar e carregar pacotes corretamente.
  • Criar projetos e organizar pastas de forma reprodutível.
NotaPré-requisitos
  • Ter a linguagem R instalado no computador.
  • Acesso à internet para baixar e instalar o RStudio
  • Acesso à internet para instalar pacotes quando necessário.
  • Noções básicas de organização de arquivos e pastas.

6.1 Instalação do RStudio

NotaInstalação do RStudio no Windows
  1. Acesse https://posit.co/download/rstudio-desktop/
  2. Role a página; normalmente o site detecta seu sistema e disponibiliza o instalador logo abaixo de 2: Install RStudio.
  3. Clique em Download RStudio Desktop for Windows se o sistema for Windows.
  4. Finalizado o download, clique duas vezes no arquivo e avance até concluir.
NotamacOS/Linux

Na mesma página de download, escolha o instalador do seu sistema. O fluxo é análogo ao do Windows: baixar, executar e concluir a instalação.

Com o RStudio instalado, vamos entender o que ele é e por que um IDE facilita o trabalho de quem está começando.

6.2 O que é o RStudio

O RStudio é um Ambiente de Desenvolvimento Integrado (IDE) voltado para a linguagem R. Em vez de abrir programas separados para escrever, executar e visualizar resultados, o IDE reúne tudo em uma única interface.

NotaConceito-chave: IDE

Um IDE integra editor de texto, execução de código, ajuda, visualização de gráficos e organização de projetos. Isso reduz erros e acelera o aprendizado.

Para quem está começando, a vantagem prática é simples: você escreve um script, executa no console e vê o resultado sem sair do mesmo ambiente.

6.2.1 Por que usar o RStudio?

  • Interface Amigável: O RStudio oferece uma interface gráfica intuitiva que facilita o trabalho, especialmente para iniciantes, com acesso visual a scripts, console, dados e gráficos simultaneamente.

  • Aumento de Produtividade: Recursos como autocompletar código, atalhos de teclado e visualização integrada de gráficos aceleram significativamente o fluxo de trabalho.

  • Gestão de Projetos: O sistema de projetos do RStudio organiza seu trabalho, mantendo todos os arquivos relacionados em um único local.

  • Integração com outras ferramentas: Oferece suporte a várias linguagens (R, Python, SQL), integra-se com sistemas de controle de versão como Git, e facilita a publicação de conteúdo via sistema Quarto.

  • Reprodutibilidade Científica: Facilita a criação de documentos reproduzíveis que combinam código, resultados e texto narrativo, essenciais para pesquisa científica e relatórios corporativos.

  • Comunidade Ativa: Desenvolvido pela Posit (anteriormente RStudio), conta com uma grande comunidade de usuários e desenvolvedores que constantemente melhoram a ferramenta.

6.2.2 Erros comuns e como resolver

  • Confundir RStudio com a linguagem R: lembre que são ferramentas diferentes.
  • Abrir o RStudio sem ter o R instalado: instale o R antes do IDE.
  • Ignorar o painel de ajuda: use a guia Help para consultar funções.

6.2.3 Checklist do leitor

6.2.4 R (linguagem) vs RStudio (IDE)

R é uma linguagem de programação, enquanto o RStudio é um Ambiente de Desenvolvimento Integrado que contém recursos que facilitam o uso da linguagem R.

Utilizando a analogia de um carro, R seria o motor enquanto o RStudio seria o painel de controles (dashboard). Portanto, é necessário ter os dois softwares instalados.

6.2.5 Erros comuns e como resolver

  • Instalar apenas o RStudio e esperar que o R funcione: instale ambos.
  • Achar que o RStudio “substitui” o R: ele apenas organiza o trabalho.
  • Não saber onde digitar comandos: use o painel Console para executar.

6.2.6 Checklist do leitor

6.3 RStudio: Uma Visão Geral

6.3.1 Interface do RStudio

Na primeira abertura, você verá quatro painéis. Eles funcionam em conjunto para escrever, executar, inspecionar objetos e visualizar resultados.

Na figura acima, cada painel tem um papel. Vamos percorrê-los começando pelo Source, que é onde você escreve.

6.3.2 Painel: Source

  • O painel Source permite a visualização e edição de vários arquivos relacionados a diversas linguagens, tais como: .R, .qmd, ou arquivos de texto gerais como .txt ou .md.

  • Por padrão, é o painel superior esquerdo e pode ser iniciado abrindo qualquer arquivo editável no RStudio.

  • Cada arquivo adicional que for aberto será adicionado como uma nova guia no painel.

6.3.3 Painel: Console

  • Esse painel fornece uma área para executar o código interativamente, isto é, você pode “conversar” diretamente com o interpretador da linguagem R.

  • Por padrão, está vinculado à linguagem R, mas ele também pode fornecer um console Python.

6.3.4 Painel: Environment

Guia Environment

  • Essa guia exibe objetos R e Python ativos na sessão atual.

  • A barra de menus de Environment fornece carregamento ou salvamento de R workspaces, importação interativa de conjuntos de dados de arquivos de texto, Excel ou SPSS/SAS/Stata.

  • Ela também exibe a memória usada pela sessão R ativa e um ícone de uma vassoura para remover todos os objetos ativos.

6.3.5 Painel: Output

Por padrão, o painel Output é o painel inferior direito e contém as seguintes guias:

Guia Files

  • Esta guia fornece uma exploração interativa da pasta de trabalho (working directory), ou seja, a pasta para a qual o R está direcionado.

  • Nesta guia, você pode adicionar, excluir, renomear e remover arquivos e pastas, entre outras operações.

Guia Help

  • O ícone Home exibe a página de ajuda geral, com links para Manuais e Referências sobre o R e RStudio.

  • Exibe também a documentação dos pacotes.

Guia Viewer

  • Exibe conteúdos interativos tais como: gráficos interativos, aplicativos Shiny, páginas HTML geradas pelo Quarto, entre outros.

Guia Presentation

  • Esta guia é usada para exibir slides HTML gerados pelo sistema Quarto.

Guia Plots

  • Esta guia exibe imagens estáticas produzidas na sessão.

  • Há setas para navegar entre gráficos mais antigos e mais recentes, zoom, um botão de exportação para salvar as imagens, um botão de exclusão de imagens e um ícone de vassoura que limpará todos os gráficos.

Guia Packages

  • Esta guia permite visualizar os pacotes R instalados.

  • Há uma barra para pesquisar a biblioteca atual de pacotes.

  • Há também um botão para instalação e atualização de pacotes.

6.3.6 Erros comuns e como resolver

  • Não saber onde digitar comandos: use o Console para executar.
  • Abrir um arquivo e não ver no Source: confirme se ele é editável.
  • Salvar arquivos fora do projeto: trabalhe sempre dentro da pasta do projeto.
  • Confundir Files com o sistema inteiro: ele mostra o working directory.

6.3.7 Checklist do leitor

6.4 Instalando Pacotes R:

6.4.1 Usando a Guia Packages

Para instalar pacotes usando a guia Packages no RStudio:

  • Clique na guia Packages.

  • Clique em Install Packages.

  • Digite o nome de cada pacote que deseja instalar separados por um espaço ou vírgula na caixa de texto.

  • Copie os pacotes abaixo e cole na caixa de texto:

tidyverse usethis
  • Clique em Install para iniciar a instalação

6.4.2 Usando comandos

  • Uma forma alternativa de instalar pacotes é usando a linha de comando.

  • Você pode usar o seguinte comando diretamente no console, em um script, ou em um documento Quarto:

install.packages(c("tidyverse", "usethis"))
NotaAtenção! Instalação e carregamento de pacotes

  • Em um computador, você instala o pacote uma única vez.

  • Para usar o pacote em cada sessão, é preciso carregá-lo com library().

# carrega os pacotes utilizados
library(tidyverse)
library(usethis)
.
.
DicaConexão com a internet

  • Para instalar pacotes de R ou Python, é necessária uma conexão ativa com a internet.

  • Em caso de falhas de conexão, o download pode falhar. Nesse caso, repita a instalação.

6.4.3 Erros comuns e como resolver

  • Tentar usar library() sem instalar: rode install.packages() antes.
  • Esperar que a instalação funcione sem internet: verifique a conexão.
  • Instalar toda vez que abrir o RStudio: instale uma vez, carregue sempre.

6.4.4 Checklist do leitor

6.5 Configurações Básicas Recomendadas do RStudio

Estas opções evitam mensagens e comportamentos confusos ao iniciar o RStudio em diferentes projetos.

  1. Abra o RStudio.

  2. Clique no menu “Tools” > “Global Options” > “Basic”.

  3. Desmarque a opção: “Restore most recently opened project at startup”.

  4. Desmarque a opção: “Restore previously open source document at startup”.

  5. Desmarque a opção: “Restore .Rdata into workspace at startup”.

  6. Na caixa à direita de “Save workspace to .Rdata on exit:”, selecione “never”

  7. Clique em “Ok” e, em seguida, em “Apply”.

6.5.1 Checklist do leitor

7 Projetos RStudio e reprodutibilidade

Depois de conhecer a interface do RStudio, o próximo passo para trabalhar de forma organizada e segura é estruturar o trabalho em projetos. Projetos reduzem confusões com arquivos, facilitam a colaboração e tornam análises mais reprodutíveis ao longo do tempo.

Em termos práticos, um projeto reúne, em uma única pasta, todos os elementos relacionados a uma análise: dados, scripts, relatórios e documentação. Essa prática é amplamente adotada por analistas e cientistas de dados e é suportada nativamente pelo RStudio por meio dos RStudio Projects.

NotaVisão geral

  • Em projetos de análise de dados, todos os arquivos relevantes ficam organizados em uma única pasta, estruturada em subpastas bem definidas (por exemplo, dados, scripts e relatórios).

  • Essa organização é tão comum que o RStudio oferece suporte direto por meio dos Projects, permitindo trabalhar com diferentes projetos de forma isolada e consistente.

  • Ao ativar um projeto, a pasta do projeto passa a ser o diretório de trabalho (working directory) da sessão, o que facilita o uso de caminhos relativos.

  • O uso de caminhos relativos reduz erros, aumenta a portabilidade do código e é um elemento central da reprodutibilidade.

7.1 Práticas ruins e como resolver

  • Trabalhar sem projeto e “perder” arquivos: crie um projeto para cada análise.
  • Misturar arquivos de projetos diferentes: use uma pasta por projeto.
  • Utilizar caminhos absolutos longos: prefira caminhos relativos dentro do projeto.

7.1.1 Checklist do leitor

7.2 Criando projetos no RStudio

Projetos RStudio estão sempre associados a uma pasta no sistema de arquivos. Há duas formas principais de criar um projeto:

  • Em uma nova pasta, que ainda não existe.
  • Em uma pasta existente, como um repositório previamente clonado do GitHub (recomendado).

É importante destacar que um projeto RStudio é conceitualmente distinto do controle de versão com Git. Um projeto pode existir sem Git, mas, neste livro, recomenda-se fortemente que projetos sejam versionados, pois isso reforça boas práticas profissionais.

Suponha que um aluno hipotético, João Silva, deseje criar um projeto para a disciplina Introdução à Ciência de Dados. Um nome adequado para a pasta e para o projeto seria joaosilva_icd, evitando espaços, acentos e caracteres especiais.

A seguir, descreve-se o procedimento para criar um projeto a partir de uma pasta existente (por exemplo, um repositório clonado do GitHub):

  1. Clique no menu Project e, em seguida, em New Project, no canto superior direito do RStudio.
  1. Na janela seguinte, selecione a opção Existing Directory.

  1. Clique em Browse e selecione a pasta que servirá como diretório do projeto (por exemplo, a pasta clonada do GitHub).

  2. Confirme a criação do projeto clicando em Create Project.

Após esse procedimento, o RStudio criará o arquivo .Rproj dentro da pasta selecionada e ativará automaticamente o projeto.

7.2.1 Erros comuns e como resolver

  • Criar projetos com nomes confusos ou inconsistentes: adote um padrão simples.
  • Salvar arquivos fora da pasta do projeto: verifique sempre a guia Files.
  • Abrir o RStudio sem ativar o projeto correto: selecione o projeto no menu Project.

7.2.2 Checklist do leitor

7.3 O que acontece quando um projeto é criado

Quando um novo projeto é criado ou ativado, o RStudio:

  • Cria um arquivo .Rproj dentro da pasta do projeto.
  • Define essa pasta como diretório de trabalho da sessão.
  • Isola o ambiente de trabalho em relação a outros projetos.

A partir desse ponto, todos os arquivos criados, scripts executados e relatórios gerados devem estar contidos na pasta do projeto.

7.3.1 Erros comuns e como resolver

  • Trabalhar fora de um projeto ativo: confirme o nome do projeto no menu Project.
  • Manter arquivos soltos fora da pasta do projeto: mova-os para a estrutura adequada.
  • Ignorar o arquivo .Rproj: ele define qual pasta corresponde ao projeto.

7.3.2 Checklist do leitor

7.4 Estrutura recomendada para o projeto RStudio criado

Para organizar adequadamente os arquivos de projetos de análise de dados, recomenda-se adotar, desde o início, uma estrutura clara e consistente de pastas. Essa organização facilita a reprodutibilidade, a colaboração e a manutenção do trabalho ao longo do tempo, além de tornar mais explícitas as etapas do processo analítico.

Neste livro, adotamos uma estrutura inicial simples, mas alinhada a boas práticas amplamente utilizadas em projetos profissionais e acadêmicos de Ciência de Dados. Essa estrutura serve como ponto de partida e pode ser expandida conforme a complexidade do projeto aumenta.

7.4.1 Estrutura básica do projeto

Inicialmente, criamos as seguintes pastas principais:

  • dados/: destinada ao armazenamento dos conjuntos de dados do projeto.
  • relatorios/: destinada aos relatórios produzidos com Quarto.
  • scripts/: destinada aos scripts R utilizados nas análises.

Dentro da pasta dados/, criamos duas subpastas com papéis bem definidos:

  • dados_brutos/: para armazenar os dados originais, sem qualquer modificação.
  • dados_limpos/: para armazenar os dados após limpeza, tratamento e transformações.

Essa separação explícita evita que dados originais sejam sobrescritos, documenta o fluxo de preparação dos dados e contribui diretamente para a reprodutibilidade das análises.

Essa estrutura será utilizada de forma consistente nos exercícios e nos exemplos apresentados ao longo do livro, servindo como referência inicial para a organização dos projetos desenvolvidos pelo leitor.

Considerando o projeto hipotético de João Silva para a disciplina Introdução à Ciência de Dados em 2026, a estrutura inicial do projeto ficaria da seguinte forma:

joaosilva_icd2026/
├── dados/
│   ├── dados_brutos/
│   └── dados_limpos/
├── relatorios/
├── scripts/
├── README.md
└── joaosilva_icd2026.Rproj

Essa organização também favorece o trabalho em equipe, pois permite que outras pessoas compreendam rapidamente onde encontrar dados, scripts e relatórios, reduzindo ambiguidades e erros operacionais.

7.4.2 Criando pastas pelo RStudio

As pastas podem ser criadas diretamente pela interface gráfica do RStudio. Na guia Files, utilize o botão New Folder para criar as pastas necessárias e navegue entre os diretórios conforme indicado.

Dica

Além da interface gráfica, é possível criar pastas e arquivos utilizando funções da própria linguagem R, como dir.create(). Essa abordagem permite documentar explicitamente a estrutura do projeto no código e reproduzi-la automaticamente em outros ambientes, sendo especialmente útil em projetos colaborativos ou versionados com Git.

7.4.3 Erros comuns e como resolver

  • Misturar dados brutos com dados limpos: utilize subpastas separadas.
  • Salvar relatórios fora da pasta relatorios/: mantenha um padrão claro.
  • Criar pastas com espaços, acentos ou caracteres especiais: prefira nomes simples e consistentes.

7.6 Configurações recomendadas (Windows)

Se você usa Windows e deseja o Git Bash como terminal no RStudio, siga as etapas abaixo. Em macOS e Linux, o terminal padrão já é suficiente.

NotaTerminal Git Bash no RStudio (Windows)

Como o RStudio e o sistema Quarto fazem parte do fluxo de trabalho, é importante saber como acessar e usar o terminal Git Bash diretamente dentro do RStudio:

  1. Configurando o terminal Git Bash no RStudio:

  • Abra o RStudio

  • Vá para “Tools” > “Global Options” > “Terminal”.

  • No campo “New terminals open with:”, selecione “Git Bash” no menu suspenso.

  • Clique em “Apply” e depois em “OK”

  1. Acessando o terminal Git Bash no RStudio:

  • No painel inferior do RStudio, clique na guia “Terminal”.

  • Um novo terminal Git Bash será aberto.

  • Se um terminal já estiver aberto, mas não for o Git Bash, clique no ícone “+” no canto superior direito do painel Terminal e selecione “New Terminal” para abrir um novo terminal Git Bash

  1. Vantagens de usar o Git Bash dentro do RStudio:

  • Mantém seu fluxo de trabalho dentro de um único ambiente.

  • Facilita a navegação entre edição de arquivos e execução de comandos Git.

  • Permite copiar e colar facilmente entre o editor e o terminal.

  • Evita a necessidade de alternar entre aplicativos diferentes.

7.6.1 Erros comuns e como resolver

  • Selecionar o terminal errado: verifique o campo “New terminals open with:”.
  • Abrir um terminal antigo: use o botão “+” para abrir um novo.
  • Esperar esse ajuste em macOS/Linux: nesses sistemas, o terminal já é padrão.

8 Atalhos essenciais

Console (Windows/Linux | macOS | Função)

  • Ctrl+L | ⌘+L | Limpar o console

8.0.1 Erros comuns e como resolver

  • Tentar memorizar tudo de uma vez: comece pelos de navegação e execução.
  • Usar atalhos sem saber o foco ativo: clique primeiro no painel desejado.
  • Ignorar a lista completa: use Alt+Shift+K/Option+Shift+K.

RStudio é um Ambiente de Desenvolvimento Integrado que organiza e facilita o uso de R, oferecendo editor de código, console, visualização de objetos e resultados em um único ambiente.

  1. Defina o que é um IDE e dê um exemplo do fluxo “escrever -> executar -> ver resultado”.
DicaReposta

Um IDE integra ferramentas para edição, execução e visualização de código em um único ambiente. No RStudio, o fluxo típico consiste em escrever o código no painel Source, executá-lo no Console e visualizar os resultados no painel Output.

  1. Explique quando usar install.packages() e quando usar library().
DicaReposta

A função install.packages() é utilizada para instalar um pacote no computador e precisa ser executada apenas uma vez. Já a função library() carrega o pacote instalado na sessão atual do R e deve ser executada sempre que uma nova sessão é iniciada.

  1. Ajuste as configurações básicas recomendadas do RStudio (restore/save) e registre, em 3 a 5 linhas, por que essas opções ajudam em projetos.
DicaReposta

Desativar a restauração automática do ambiente e o salvamento do arquivo .RData torna o comportamento das sessões mais previsível. Essas opções reduzem dependências implícitas do ambiente anterior, facilitam a reprodução das análises e evitam erros causados por objetos carregados automaticamente sem controle explícito do código.

  1. Crie um novo projeto no RStudio com um nome padronizado (sem espaços) e explique o que é um caminho relativo e por que ele facilita a reprodutibilidade.
DicaReposta

Caminhos relativos referenciam arquivos a partir da pasta raiz do projeto, em vez de utilizar um endereço fixo no computador. Isso permite que o mesmo projeto funcione corretamente em diferentes máquinas e sistemas operacionais, facilitando o trabalho em equipe, a reprodutibilidade e o uso de controle de versão com Git e GitHub.
